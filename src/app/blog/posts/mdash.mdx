## Contexte

WordPress propulse encore plus de 40 % des sites web dans le monde, mais son architecture — un cœur PHP monolithique où plugins et thème s'exécutent dans le même processus que le noyau — accuse aujourd'hui ses limites. La majorité des failles de sécurité WordPress proviennent justement de cette absence d'isolation entre extensions.

C'est dans ce contexte que Cloudflare a lancé en avril 2026 **EmDash**, un CMS open source écrit entièrement en **TypeScript**, construit sur **Astro**, et pensé comme un "successeur spirituel" de WordPress plutôt qu'un simple clone.

## Installation

L'installation se fait via la CLI officielle, en local avec SQLite ou directement sur l'infrastructure Cloudflare avec D1 et R2.

## Architecture

La différence structurante avec WordPress se situe dans l'exécution des plugins. Chaque extension EmDash tourne dans un **Worker isolé** (Dynamic Worker Loader de Cloudflare) et doit déclarer explicitement ses permissions via un manifeste de capacités. Sans déclaration d'accès en écriture, par exemple, un plugin ne peut tout simplement pas modifier le contenu — même s'il est compromis.

<CodeBlock
    marginBottom="16"
    highlight="2-7"
    codes={[
  {
    code:
`export default definePlugin({
  id: "comments",
  capabilities: ["read:content", "write:comments"],
  hooks: {
    "content:afterSave": async (event, ctx) => {
      // Ne peut agir que dans le périmètre déclaré
    }
  }
});`,
    language: "ts",
    label: "Déclaration d'un plugin EmDash avec capacités"
  }
]} />

Autre choix notable : le contenu n'est pas stocké en HTML mais en **Portable Text**, un format JSON structuré. Cela permet de réutiliser le même contenu pour un site web, une app mobile ou une API, sans avoir à parser du DOM.

## Mon point de vue

**Modernité.** TypeScript de bout en bout, types générés automatiquement depuis le schéma de la base : le confort de développement est réel pour quiconque vient de Next.js ou d'Astro.

**Sécurité.** L'isolation par Worker change vraiment la donne face au modèle WordPress, où un plugin compromis a accès à toute la base de données.

**Pensé pour les agents IA.** EmDash embarque un serveur MCP natif, ce qui permet à un outil comme Claude de lire, modifier et faire évoluer le schéma de contenu directement — sans passer par une API REST custom.

**Écosystème jeune.** EmDash est encore en bêta (v0.1) au moment de la rédaction. La marketplace de plugins est embryonnaire comparée aux dizaines de milliers d'extensions WordPress, et certains cas d'usage avancés (multisite, e-commerce complexe) ne sont pas encore couverts.

## Conclusion

EmDash n'a pas vocation à remplacer WordPress du jour au lendemain, mais il pose une question légitime : à quoi ressemblerait un CMS conçu en 2026, avec les standards de sécurité et les usages IA actuels ? Pour une stack déjà orientée Astro/TypeScript, c'est une option sérieuse à surveiller de près.
